# PlotWeaver 开发与测试流程

本文档记录 PlotWeaver 从需求分析、结构设计、后端实现、前端联调到测试验证的完整工程流程。

## 1. 需求分析

题目要求开发一款 AI 小说转剧本工具，核心目标是：

- 接收 3 个章节以上的小说文本。
- 自动转换为结构化剧本。
- 输出 YAML 格式，便于作者继续编辑和打磨。
- 额外提供 YAML Schema 文档，并说明 Schema 的设计原因。

结合比赛交付周期，v1 范围确定为：

- 支持 TXT、DOCX、EPUB 上传。
- PDF/OCR 暂不支持。
- 不支持带 DRM 加密的 EPUB。
- 单文件中包含多章节小说。
- 后端负责文件解析、章节识别、AI 调用、结构校验和密钥保护。
- 前端负责上传、进度展示、ScreenYAML 预览、剧本预览、黑白主题切换和 YAML/TXT 下载。

## 2. 技术选型

### 后端

- FastAPI：快速实现上传接口、校验接口和 SSE 流式接口。
- Pydantic v2：定义 LLM Draft 与 ScreenYAML 数据模型。
- PyYAML：序列化最终 YAML。
- python-docx：提取 DOCX 文本。
- EbookLib + BeautifulSoup：按 EPUB spine 顺序提取章节正文。
- OpenAI SDK：调用 OpenAI-compatible Chat Completions API。

### 前端

- React + Vite：快速构建本地 Web 工作台。
- TypeScript：约束 SSE 事件和页面状态。
- Lucide React：提供上传、下载、状态提示等图标。

### 未采用方案

- LangChain / LangGraph：v1 不采用，避免在短周期内引入额外调试成本。
- Token 级 JSON 流式解析：不采用。系统只流式传输已经通过后端处理的稳定事件。

## 3. Schema 设计流程

项目确定使用 ScreenYAML v2.3 作为最终结构化输出格式。

核心设计原则：

- ScreenYAML 是唯一数据源。
- LLM 只输出草稿，不掌握最终 `character_id`。
- 后端负责角色 ID 主权、动作去主语和字段归一化。
- 正文块不使用 UUID，减少冗余。
- 场景采用扁平 `scenes` 数组，方便按章节流式追加。

正文块分为四类：

- `environment`：环境、物件、声音、空间氛围，无人物主体。
- `action`：某个角色的无台词动作。
- `performance`：同一角色的“说话前动作 + 台词 + 说话后动作”。
- `transition`：转场提示。

相关文档：

- [ScreenYAML v2.3 Schema 规范](./ScreenYAML-v2.3-schema.md)
- [C-Fountain v1 渲染与导出规则](./C-Fountain-v1-rendering.md)

## 4. 后端实现流程

### 4.1 文件解析

文件：`backend/app/chapter_parser.py`

实现内容：

- 支持 `.txt`、`.docx`、`.epub`。
- `.txt` 按 UTF-8 优先，失败后尝试常见中文编码。
- `.docx` 使用 `python-docx` 提取段落文本。
- `.epub` 使用 EbookLib 读取 spine 文档，并用 BeautifulSoup 提取 HTML 正文。
- `.pdf` 直接拒绝，并提示 v1 暂不支持。
- 章节识别支持：
  - `第1章`
  - `第一章`
  - `Chapter 1`
  - `CHAPTER ONE`
- 少于 3 章时返回系统要求提示。

### 4.2 数据模型

文件：`backend/app/models.py`

模型分为两层：

- `LLMChapterDraft` / `LLMSceneDraft`：模型输出草稿。
- `ScreenYAML` / `Scene` / `BodyBlock`：最终结构化输出。

这样可以避免让大模型直接生成最终 ID 和最终 YAML。

### 4.3 草稿适配

文件：`backend/app/draft_adapter.py`

作用：

- 兼容模型偶尔输出的字段变体。
- 将 `dialogue/content/speaker` 等字段适配为标准 LLM Draft。
- 降低因为字段名轻微漂移导致整章转换失败的概率。

### 4.4 归一化

文件：`backend/app/normalizer.py`

实现内容：

- 后端维护角色注册表。
- 根据 `name` 和 `aliases` 复用或创建角色 ID。
- 自动汇总 `characters_present`。
- 对动作文本执行“去主语”处理。
- 角色匹配不可靠时保留可读文本，并可标记 `needs_review`。

### 4.5 AI 调用

文件：

- `backend/app/llm_client.py`
- `backend/app/prompt_templates.py`

实现内容：

- 使用 OpenAI-compatible Chat Completions API。
- 支持 `OPENAI_RESPONSE_FORMAT=json_object` 或 `none`。
- 模型返回 markdown JSON 代码块时尝试提取其中 JSON。
- Prompt 明确要求输出 LLM Draft JSON，不输出最终 YAML。
- Prompt 约束动作、环境、台词和电话对白的判断规则。

### 4.6 流式转换

文件：

- `backend/app/converter.py`
- `backend/app/sse.py`

流程：

1. 后端解析上传文件并识别章节。
2. 按章节顺序调用模型。
3. 将模型返回数据适配为 LLM Draft。
4. 使用 Pydantic 校验草稿结构。
5. 使用 normalizer 生成最终 Scene。
6. 每完成一个 scene，通过 SSE 输出 `scene_completed`。
7. 全部完成后输出 `validation_result` 和 `final_yaml`。

SSE 事件：

- `progress`
- `chapter_started`
- `scene_completed`
- `validation_result`
- `final_yaml`
- `error`
- `done`

### 4.7 渲染与校验

文件：

- `backend/app/yaml_service.py`
- `backend/app/validator.py`

实现内容：

- ScreenYAML 序列化。
- 单场景 YAML 片段输出。
- C-Fountain v1 渲染。
- 中文两行场景头：

```text
第 {scene_index} 场 （第 {source_chapter_index} 章）
{camera_location} / {setting} / {time_of_day}
```

## 5. 前端实现流程

文件：

- `frontend/src/main.tsx`
- `frontend/src/styles.css`

实现内容：

- 点击或拖拽上传小说。
- 本地预检查文件类型和大小。
- 调用 `/api/upload` 预检章节数。
- 调用 `/api/convert/stream` 接收 SSE。
- `scene_completed` 到达时立即追加 ScreenYAML 片段和结构化剧本预览。
- `final_yaml` 到达时替换为完整最终文件内容，确保下载完整。
- 显示章节进度、场景数和错误信息。
- 生成过程中支持停止生成，前端会立即断开 SSE/fetch 流并保留已生成结果。
- 剧本预览支持黑底和柔和白底两种主题。
- 剧本预览可导出为 TXT；ScreenYAML 预览可导出为 YAML。

## 6. 启动流程

### 后端

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app.main:app --reload
```

### 前端

```powershell
cd frontend
npm install
npm run dev
```

访问：

```text
http://localhost:5173
```

## 7. 测试流程

### 后端测试

```powershell
cd backend
$env:PYTHONPATH='.'
python -m pytest -o cache_dir=$env:TEMP\plotweaver-pytest-cache tests
```

当前测试覆盖：

- 中文、英文和 EPUB 章节识别。
- LLM Draft 字段变体适配。
- 角色 ID 稳定分配。
- 动作文本去主语。
- 未知角色被字句动作降级为环境描写。
- 单场景 C-Fountain 渲染。

### 前端类型检查

```powershell
cd frontend
npx tsc -b
```

### 前端构建

```powershell
cd frontend
npm run build
```

当前环境曾出现 `spawn EPERM`，表现为 Vite 调用 esbuild 子进程时被 Windows 权限策略或安全软件拦截。此问题不属于 TypeScript 类型错误，前端类型检查已通过。

## 8. 手工联调流程

1. 启动后端。
2. 启动前端。
3. 打开 `http://localhost:5173`。
4. 上传包含 3 章以上的 TXT、DOCX 或 EPUB 小说。
5. 确认文件分析区显示章节数。
6. 点击“生成结构化剧本”。
7. 观察章节进度、场景数和实时追加的 ScreenYAML/剧本预览片段。
8. 等待最终 `final_yaml` 到达。
9. 下载 `.yaml` 和 `.txt` 文件。
10. 检查生成内容是否符合 ScreenYAML Schema 和剧本预览规则。

## 9. 已知限制

- v1 不支持 PDF 和 OCR。
- v1 不支持带 DRM 加密的 EPUB。
- 模型输出质量依赖所选模型和 Prompt 遵循能力。
- 长篇小说会产生较长等待时间，但按场景流式输出可以提前看到结果。
- 不同 OpenAI-compatible 服务对 `response_format` 支持不同；不支持时应设置 `OPENAI_RESPONSE_FORMAT=none`。
- C-Fountain v1 是中文剧本友好的 Fountain-compatible 导出规则，不等同于完整英文 Fountain 工业规范。

## 10. 后续可扩展方向

- 增加 YAML 编辑器和实时行号错误定位。
- 增加角色表可视化编辑。
- 增加 `needs_review` 前端高亮与人工修正。
- 支持更细粒度的场景来源定位。
- 增加 PDF 文本提取或 OCR。
- 增加模型输出重试与局部修复机制。
